fn alloc_safe_slice<T: SupportedSliceType + Debug>( &mut self, obj: &[T], ) -> GcSlice<T>

Allocates a slice that only contains values that ARE NOT pointers. Allocating a Vec is fine, allocating a Vec is fine if they’re all Integer values. Not unforced by the Rust type system atm, but we could make some nice traits for this. Just afraid that this would add unnecessary complexity.

Deprecated because too likely to be unsafe: GC triggered when allocating a slice makes the slice likely to be invalid. Now every uses should be replaced with alloc_safe_slice, or with request_mem_for_slice + write_slice_to_addr

fn request_bytes( &mut self, size: usize, _alloc_origin_marker: Option<AllocSiteMarker>, ) -> Address

Request size bytes from MMTk. Importantly, this MAY TRIGGER A COLLECTION. Which means any function that relies on it must be mindful of this, such as by making sure no arguments are dangling on the Rust stack away from the GC’s reach.
